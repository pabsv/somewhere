## Architectural Decisions

- **Ground-competitive destination cap (2026-07-19):** destinations close enough to the NL/BE origins that a train/Flixbus is a real alternative only surface on the discovery boards when the whole round trip is тЙд `GROUND_COMPETITIVE_MAX_PRICE` (тВм100) тАФ above it the ground option wins and the flight was just clutter (user reported Paris / near-German listings as "stupid"). A genuinely cheap one (e.g. a sub-тВм100 Berlin steal) still shows, per the user's "take it if it's a steal" rule. Pieces in `lib/score.ts`: `GROUND_COMPETITIVE_CODES` (set тАФ **all Germany** BER/MUC/FRA/HAM/STR/HAJ/NUE/LEJ/DRS/BRE/FMM/FKB/FMO + Luxembourg LUX + **northern France** CDG/ORY/BVA/LIL; southern France / Switzerland / UK deliberately uncapped тАФ real flights), `GROUND_COMPETITIVE_MAX_PRICE`, `isGroundCompetitive()`. Enforced in `lib/queries.ts` via one `GROUND_COMPETITIVE_NOR` Mongo `$nor` clause ("NOT (ground-competitive AND >cap)", composes without touching the existing `price` key) merged into **`buildTripFilter`** (Calendar bars+density, DaySheet, Group calendar) and the **`getCitiesData`** match (Explore grid, FavouritesStrip, landing DepartureBoard). **City drill-in (`getCityData`) is EXEMPT** тАФ a deliberate `/city/[code]` visit / SearchCombobox pick shows the full picture; the cap only de-clutters the boards. So a capped-out city (no sub-тВм100 fare) vanishes from Explore/Calendar but stays reachable by search. Verified live 2026-07-19: 170/170 current ground-competitive future flights are >тВм100 (cheapest BER тВм120), so all are currently hidden. Tuning = one constant / edit the set. Follow-ups not built: (1) let a **favourited** city bypass the cap (user explicitly wants it) тАФ needs per-user set in the query; (2) apply to open-jaw/twin combos (currently exempt тАФ niche + already curated).

- **Pool scraping (2026-05-28):** decoupled scraping from users. Pool of 1150 (origin, dest) routes lives in `scrape_targets`, tier A/B/C cadenced (24h / 72h / 168h). Scheduler claims one route per 2-min slot, 24h (widened from 07-23 on 2026-07-14 тАФ box runs always-on). Users will be matched to the pool later, but scraping does not depend on user prefs.
- **Origins (5):** EIN, AMS, BRU, CRL, MST тАФ airports closest to Eindhoven with usable coverage. MST (Maastricht) added 2026-07-14 by user request; small airport, so most of its routes will come back empty and auto-disable. DUS / NRN commented out in `targets.py` for easy re-add.
- **Currency pinned to EUR (2026-07-14):** Google Flights picks the response currency itself (GeoIP) unless the request URL pins it тАФ `?gl=NL&hl=en&curr=EUR` is appended to both fli endpoints in `FliScraper.__init__`. Before the pin, ~30% of stored flights were GBP/DKK/CZK/HKD/... with their raw numbers treated as EUR. A save-time guard in `flight_service.py` drops any non-EUR fare as defense in depth. Legacy non-EUR docs age out via the 14d flights TTL.
- **Destinations (230):** Europe + N.Africa + Levant + T├╝rkiye + Atlantic. Curated in `scraper/targets.py`. Tier A=28 popular, B=84 mid, C=118 long-tail.
- **Per-route scrape recipe (2026-07-17 тАФ any-duration one-way sweeps):** Phase 1 = 2 one-way SearchDates sweeps (OтЖТD over 181d тЙИ 3 HTTP calls after fli's internal 61-day chunking; DтЖТO over ~189d тЙИ 4 calls); every (out, ret) pair with 2тАУ10 nights combined in memory (estimated price = sum of one-ways). Phase 2 = top 10 cheapest pairs (max 2 per departure date тАФ `SCRAPE_MAX_PER_OUT_DATE`) via SearchFlights, real round-trip prices stored. тЙИ17 fli HTTP calls per route per cycle. If one-way grids come back empty, falls back to round-trip grids (durations 2/6/10) so the route doesn't count toward auto-disable. Knobs: `SCRAPE_MIN_NIGHTS` / `SCRAPE_MAX_NIGHTS` / `SCRAPE_TOP_N_CHEAP_DATES` in `database/config.py` (replaced `SCRAPE_DURATION_BUCKETS`).
- **One-way fare grids / open-jaw foundation (2026-07-17):** the Phase-1 one-way SearchDates grids (both directions, previously discarded after pair ranking) are now persisted to a new `oneway_fares` collection тАФ **one doc per directed leg** (`leg_key` = `"EIN-BCN"`, `prices: {date: cheapest one-way тВм}` replaced wholesale each scrape, ~2300 docs тЙИ 12MB, TTL 21d on `scraped_at`). Zero extra API calls. Pieces: `database/models/oneway_fare.py`, `database/repositories/oneway_fare_repo.py`, `FlightService.save_oneway_grids` (past-date + price-sanity filter; empty result skips the leg so the old grid survives), scraper exposes grids via `stats["oneway_grids"]`, pool scheduler saves best-effort/non-fatal (skipped on `--direct-only`). Purpose: future open-jaw/multi-city = two `find_by_leg` lookups + in-memory date join тАФ origin-side (`EINтЖТBCN` ├Ч `BCNтЖТAMS`) and destination-side (`EINтЖТBCN` ├Ч `MADтЖТEIN`) both covered since every pool route is swept both directions; combo price = sum of one-ways = real bookable price (two tickets). No UI/read path built yet. Caveat: return-direction grids start ~today+min_nights.
- **TTL cleanup:** `flights.last_seen_at` 14d (re-scraping refreshes the touch), `scrape_runs.started_at` 30d, `price_history.scraped_at` 180d, `oneway_fares.scraped_at` 21d. Past-date trips drop naturally because SearchDates only returns future dates.
- **Auto-disable:** a route is `enabled=false` after 5 consecutive empty/error runs. `seed --prune` resets orphans; `repo.reenable_all()` resurrects disabled routes.
- **Fli backend:** `scraper-fli/scraper.py`. Single instance per slot. No NordVPN rotation yet тАФ add only if Google starts rate-limiting (DormSpot pattern).
- Azair only тАФ Kiwi dropped (too rate-limited). Azair scraper kept as fallback backend.
- Multi-user, no password тАФ identity via `X-User-ID` header (stored in localStorage). Login is name + email only.
- Frontend connects directly to Atlas via Next.js API routes (serverless) тАФ no backend HTTP server needed
- **Landing / Explore split (2026-06-13):** `/` (`app/page.tsx`) is now a promo landing page тАФ pitch headline ("Open to go anywhere, on any free day."), the live cheapest-steals `DepartureBoard`, CTA тЖТ `/explore`. `/explore` (`app/explore/page.tsx`) is the moved browsing page: board + grid. Explore sort is **cheapest-only** (no Best score / Most trips); the old region chip row + text input are replaced by one `SearchCombobox` (city / country / region typeahead, `components/explore/SearchCombobox.tsx` тЖТ `SearchSelection`); the "Only my free dates" toggle sits in the search row's trailing slot. Nav "Explore" + city back-link point at `/explore`; the wordmark points at `/`.
- **Interest cities / saved (2026-06-13):** users star destinations as focus points. Stored as `users.saved_cities: string[]` (uppercase IATA, sanitized + capped 500).
